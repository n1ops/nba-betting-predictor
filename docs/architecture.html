The model prioritizes recent performance (last 5 games = 45% weight) while anchoring to medium and long-term averages. Trend adjustment captures momentum ‚Äî a player on a hot streak gets a slight boost. Consistency score measures variance: a player who scores 20 every night is more predictable than one who alternates between 10 and 30. The 8% edge threshold filters out marginal plays, only surfacing picks where our model significantly disagrees with the market.

Technical Details

üìã DynamoDB Design

  • Pattern Single-table design with entity_type attribute
  • PK/SK Composite keys: PLAYER#id / GAME#id#date
  • GSI GSI1 for date-based queries (DATE#date ‚Üí PLAYER#id)
  • TTL 30-day auto-expiry on predictions
  • Billing On-demand (pay per request, no provisioned capacity)

üîí Security

  • API Read-only GET endpoints, no authentication required
  • IAM Least-privilege roles per Lambda (SAM-generated)
  • S3 Raw data bucket fully locked (all public access blocked)
  • Keys API keys stored in Lambda environment variables
  • Data No PII or user data stored anywhere
